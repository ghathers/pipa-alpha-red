**Architecture and Design Decisions: Post-Incident Planning App (PIPA)**

**1. Architectural Overview**
The Post-Incident Planning App (PIPA) is designed to ingest highly technical forensic data and executive summaries, synthesizing them into actionable, regulatory-compliant recovery workflows. The application utilizes a lightweight Retrieval-Augmented Generation (RAG) pipeline powered by Gemini 3.1 Pro Preview via Google AI Studio.

The architecture was chosen for its massive context window, allowing both the 200-page Alpha Red Forensic Report and the Executive Slides to be processed simultaneously without token truncation.

**2. Core Design Decisions**
Model Selection: Gemini 3.1 Pro Preview was selected over Flash due to the necessity for "Conflict Resolution Logic." The system needed the reasoning depth to identify contradictions between the highly optimistic Executive Slides and the stark reality of the Forensic Report.

Structured Output (JSON): To ensure seamless integration with the frontend UI (designed via Stitch), the AI's output was strictly constrained to a predefined JSON schema. This guarantees that UI components like the "Regulatory Risk Heatmap" always receive the correct data types.

Zero-Trust Grounding: A "Reasoning Constraint" was designed into the System Instructions. The model was instructed to treat the Forensic Report as the absolute source of truth, effectively stripping away the "Executive Bias" present in the presentation slides.

**3. System Pipeline**
Ingestion: Multimodal ingestion of Report.pdf and Slides.txt.

Analysis: The model performs a "Gap Analysis" against common regulatory frameworks (e.g., GDPR Article 33).

Synthesis: Generation of strict JSON output containing discrepancies, immediate risks, and a 60-day recovery plan.

Presentation: The JSON is rendered into a dynamic executive dashboard generated by Google Stitch.
